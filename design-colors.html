Key Improvements in New System:
  • Clearer hierarchy: Light gray background (--neutral-96) makes content more readable at rest
  • Better interactive states: Distinct hover (--blue-160) and selected states (--blue-150) provide clear feedback
  • Updated brand color: Badge color updated from #47A1C3 (legacy) to #0F6CBD (new brand blue)
  • Consistent shadows: All cards use --shadow-04; hover uses slightly elevated --shadow-light-08
  • Selected state: Light blue background (--blue-150) clearly indicates active selection

7. Timeline / Shift Colors

The new schedule uses a simple, readable color system: light blue for Base Shift, orange for Override Shift, and green for Breaks / Outputs / other auxiliary events. This replaces the old grey-heavy palette with clearer, role-based colors that are easier to scan at a glance.

Base Shift
Default working time
Override Shift
Manual or exceptional schedule
Break / Outputs
Lunch, break, meeting, warehouse, etc.

Visual Examples

Tokens

Token Purpose Default
--tl-base-bg Base Shift fill #CFE4FA
--tl-base-border Base Shift border #7ABCE6
--tl-override-bg Override Shift fill #FAD4B0
--tl-override-border Override Shift border #CB6C1F
--tl-aux-bg Break/Outputs fill #CFF0CD
--tl-aux-border Break/Outputs border #3CC1A4
--tl-chip-bg Inline chip background #FFFFFF
--tl-chip-text Inline chip text #242424
--tl-now-line Vertical/marker line #EA4632
Why this change? The old view used dense greys that made event types hard to distinguish. The new palette assigns a distinct hue per role (base/override/aux), improving at-a-glance comprehension while keeping text legible (dark text on light fills). Em português, por alto: azul = turno base; laranja = turno de override; verde = pausas/saídas.

Accessibility

  • Use dark text (--neutral-14 / #242424) on light fills to meet contrast for labels inside bars.
  • For tiny chips, keep a white chip background (--tl-chip-bg) with dark text and a subtle border for readability.
  • Do not rely on color alone—retain labels like WORK, LUNCH, WAREHOUSE inside segments.

8. Implementation Guide

Step 1: Add Design System CSS

Create a design-system.css file and include it in your project after all vendor CSS files (PrimeNG, etc.):

<!-- In your HTML head --> <link rel="stylesheet" href="vendor/primeng.css"> <link rel="stylesheet" href="styles.css"> <link rel="stylesheet" href="design-system.css"> <!-- Load last -->

Step 2: Use Design Tokens

Reference CSS variables in your stylesheets:

.my-button { background-color: var(--btn-primary-bg); color: var(--btn-primary-text); } .my-card { background: var(--card-bg); box-shadow: var(--shadow-04); }

Step 3: Apply Component Classes

Use the pre-built classes for common components:

<!-- Buttons --> <button class="btn btn--primary">Apply</button> <button class="btn btn--secondary">Cancel</button> <!-- Form Fields --> <input type="text" class="ds-input" placeholder="Subject"> <textarea class="ds-textarea" placeholder="Message"></textarea> <!-- Cards --> <div class="ds-card"> <!-- Card content --> </div>
